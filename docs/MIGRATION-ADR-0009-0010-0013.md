# Migration plan — exécution des ADRs 0009, 0010, 0013

Ces trois ADRs sont **actées** mais **pas exécutées**. Elles bloquent la
consolidation des repos de standardisation. Plan d'exécution ci-dessous.

---

## ADR-0009 — Fusion `chrysa/github-actions` → `chrysa/shared-standards/.github/actions/`

**État actuel** : `chrysa/github-actions` contient 12 composite actions
(gitversion, install-project, mypy-check, publish-python-package, python-setup,
ruff-check, run-tests, sonar-js-scan, sonar-scan, sonar-scan-node, sonar-scan-python,
tool-setup).

**Cible** : ces actions vivent sous `shared-standards/.github/actions/` et les
workflows `workflow_call` de `shared-standards/.github/workflows/` les appellent
via `uses: ./.github/actions/<name>`.

### Étapes atomiques

| ID | Action |
|---|---|
| M09-01 | Cloner `chrysa/github-actions` en local |
| M09-02 | Cloner `chrysa/shared-standards` en local |
| M09-03 | `mkdir -p shared-standards/.github/actions/` |
| M09-04 | Copier chaque action une par une (ne pas `mv` en bloc) |
| M09-05 | Vérifier que chaque `action.yml` est valide (`actionlint .github/actions/<name>/action.yml`) |
| M09-06 | Mettre à jour `workflows/ci-python.yml` : `uses: chrysa/github-actions/ruff-check@v1` → `uses: ./.github/actions/ruff-check` |
| M09-07 | Mettre à jour idem pour `ci-node.yml`, `release.yml`, `sonar.yml` |
| M09-08 | Grep dans tous les repos `chrysa/*` pour trouver les appelants `uses: chrysa/github-actions/` |
| M09-09 | Ouvrir une PR sur chaque repo appelant pour repointer vers `chrysa/shared-standards/.github/actions/<name>@main` |
| M09-10 | Merger la PR shared-standards |
| M09-11 | Merger les PRs des repos appelants |
| M09-12 | Archiver `chrysa/github-actions` (pas delete — garde l'historique) |
| M09-13 | Mettre à jour la page Notion "Ecosystem Reorg Q2 2026" → ADR-0009 status = EXÉCUTÉ |

## ADR-0013 — Fusion `chrysa/project-init` → `chrysa/shared-standards/packages/project-init/`

**Cible** : `shared-standards/packages/project-init/` contient le CLI. Expose un
entrypoint `project-init` installable via `pipx install chrysa-shared-standards`
ou lancé via `curl … | bash` (le `bootstrap-new-repo.sh` fourni par `standards-updates`).

## Ordre d'exécution global

1. **ADR-0010 d'abord** — pas de dépendance externe, pur refactoring
2. **ADR-0009 ensuite** — les workflows `workflow_call` fournis par `standards-updates` s'appuient déjà sur des composite actions relatives
3. **ADR-0013 à la fin** — la CLI project-init dépend des templates déjà consolidés dans shared-standards

## Dépendance avec le kit `standards-updates/`

Le dossier `/mnt/chrysa/standards-updates/` contient :

- Les **templates** (Dockerfile Python/Node, compose, Makefile modulaire, gitignore Python/Node, pre-commit) → à déposer dans `shared-standards/templates/`
- Les **workflows reusable** refactorés en `workflow_call` → à déposer dans `shared-standards/.github/workflows/`
- Le **bootstrap script** → à déposer dans `shared-standards/scripts/` puis déplacé dans `packages/project-init/bin/` au moment de M13

Ces livrables sont à merger **avant** M09 (car ci-python.yml refactoré en
workflow_call est la cible de la consolidation).

Les 3 ADRs sont exécutées quand :

- [ ] `chrysa/github-actions` est archivé

- [ ] `chrysa/project-init` est archivé
- [ ] Tous les repos appelant les anciens chemins ont été mis à jour et leur CI passe au vert
- [ ] La page Notion "Ecosystem Reorg Q2 2026" est à jour
- [ ] Un nouveau repo peut être créé en 1 commande : `curl … | bash -s -- --type python --name foo --apply`
